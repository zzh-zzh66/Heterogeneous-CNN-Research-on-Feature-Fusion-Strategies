## 二、什么是学习率？

**学习率（Learning Rate）** 控制模型参数更新的步长。

```
参数更新公式：
    新参数 = 旧参数 - 学习率 × 梯度

类比：
    · 学习率大 = 大步下山，容易跨过最优解附近，来回震荡
    · 学习率小 = 小步下山，精度高但收敛慢，容易卡在局部最优
```

### 本实验中的学习率变化

```
Epoch  1 ~ 27:  lr = 0.001       (初始学习率)
Epoch 28 ~ 45:  lr = 0.0005      (第1次衰减, 减半)
Epoch 46 ~ 50:  lr = 0.00025     (第2次衰减, 再减半)
```

---

## 三、学习率衰减是如何触发的？

代码中使用的是 `ReduceLROnPlateau` 调度器：

```python
scheduler = lr_scheduler.ReduceLROnPlateau(
    optimizer,
    mode='min',              # 监控 val_loss，越低越好
    patience=5,              # 连续5个epoch不下降才衰减
    factor=0.5,              # 衰减系数：lr → lr × 0.5
    min_lr=1e-6
)
scheduler.step(val_stats['loss'])  # 传入验证集损失
```

### 触发条件

`mode='min'` 意味着当验证损失 **连续 5 个 epoch 不再下降** 时，学习率自动减半。

## 四、从实验数据看学习率衰减的效果

### 衰减前：Epoch 1-27（lr=0.001）

| 阶段 | Epoch | Train Acc | Val Acc | lr | 趋势 |
|------|-------|-----------|--------|-----|------|
| 开始 | 1 | 40.54% | 43.36% | 0.001 | 快速上升 |
| 中期 | 14 | 68.02% | 66.68% | 0.001 | 波动 |
| 后期 | 27 | 72.58% | 69.76% | 0.001 | 趋于平稳 |
| 衰减触发点 | 28 | 72.94% | 69.34% | **→ 0.0005** | |

**现象**：验证准确率在 66-70% 之间波动，连续 5 个 epoch 无法突破。

### 衰减后第1次：Epoch 28-45（lr=0.0005）

| 阶段 | Epoch | Train Acc | Val Acc | lr | 趋势 |
|------|-------|-----------|--------|-----|------|
| 衰减后起点 | 28 | 74.50% | 69.34% | 0.0005 | 小步精细调整 |
| 中期 | 36 | 75.65% | 72.96% | 0.0005 | 逐步上升 |
| 后期 | 45 | 76.48% | 71.26% | 0.0005 | 再次趋于平稳 |
| 衰减触发点 | 46 | 76.61% | 73.02% | **→ 0.00025** | |

**现象**：学习率降低后，验证准确率从 69% 提升到 73%，说明小步长帮助模型找到了更优的位置。

### 衰减后第2次：Epoch 46-50（lr=0.00025）

| Epoch | Train Acc | Val Acc | lr | 趋势 |
|-------|-----------|---------|-----|------|
| 46 | 76.61% | 73.02% | 0.00025 | 继续上升 |
| 48 | 77.66% | **74.92%** ★ | 0.00025 | 达到最佳 |
| 50 | 77.75% | 74.04% | 0.00025 | 基本稳定 |

**现象**：第二次衰减后，验证准确率突破 74%，最终测试准确率达到 **74.61%**。

## 六、为什么需要学习率衰减？

### 大学习率的问题

```
大步下山，容易在最优解附近震荡：

最优解在 x=0

lr=0.1:  x=2 → x=-1 → x=0.5 → x=-0.2 → ... (来回跳动)
lr=0.01: x=2 → x=0.8 → x=0.3 → x=0.1 → ... (收敛更稳)
```

### 学习率衰减的作用

| 阶段 | 学习率 | 作用 |
|------|--------|------|
| 初期 | 大 (0.001) | 快速探索，找到大致方向 |
| 中期 | 中 (0.0005) | 精细调整，向最优解靠近 |
| 后期 | 小 (0.00025) | 微调，避免跨过最优解 |

```
学习率衰减就像下山过程：
  初期：大步快走（高学习率）→ 快速接近山脚
  中期：中步稳走（中学习率）→ 找到正确路径
  后期：小步细走（低学习率）→ 精确到达最低点
```

## 七、ReduceLROnPlateau 的工作原理

### 算法逻辑

```
每个 epoch 结束时：
    1. 计算验证指标（损失）
    2. 如果当前值 < 历史最佳 → 记录最佳值，重置计数器
    3. 否则计数器 +1
    4. 如果计数器 ≥ patience → lr = lr × factor，计数器归零
```

### 本实验中的触发过程（基于 val_loss）

```
Epoch 27: val_loss = 0.8556  ← 当前最佳，重置计数器
Epoch 28: val_loss = 1.0244  ← 上升（变差），计数器=1
Epoch 29: val_loss = 0.9324  ← 仍高于0.8556，计数器=2
Epoch 30: val_loss = 0.9324  ← 仍高于0.8556，计数器=3
Epoch 31: val_loss = 0.9324  ← 仍高于0.8556，计数器=4
Epoch 32: val_loss = 1.0244  ← 仍高于0.8556，计数器=5 → 触发衰减！
             lr: 0.001 → 0.0005

Epoch 34: val_loss = 0.7908  ← 新最佳，重置计数器
...
Epoch 41: val_loss = 0.8414  ← 仍高于0.7908，计数器=7
Epoch 42: val_loss = 0.8432  ← 仍高于0.7908，计数器=8
Epoch 43: val_loss = 0.8432  ← 仍高于0.7908，计数器=9
Epoch 44: val_loss = 0.8153  ← 仍高于0.7908，计数器=10
Epoch 45: val_loss = 0.8300  ← 仍高于0.7908，计数器=11
Epoch 46: val_loss = 0.7699  ← 新最佳，重置计数器
...
Epoch 51: val_loss = 0.7353  ← 持续下降，未触发衰减

             lr: 0.0005 → 0.00025
```

**注意**：由于 val_loss 波动大，计数器累积速度比预期慢。Epoch 28 的衰减实际等待了 5 个 epoch。Epoch 46 的衰减则等待了约 12 个 epoch 才等到新的最低点。

## 八、其他常见学习率调度策略

| 策略 | 原理 | 特点 |
|------|------|------|
| **StepLR** | 每 N 个 epoch 固定衰减 | 简单，不够灵活 |
| **ReduceLROnPlateau** | 指标停滞时衰减 | 自适应，依赖监控指标 |
| **CosineAnnealing** | 余弦曲线衰减 | 平滑，通常效果较好 |
| **Warmup + Cosine** | 先升后降 | 稳定，适合大模型 |
| **Exponential LR** | 指数衰减 | 衰减速度快 |

### 本项目选择 ReduceLROnPlateau 的原因

1. **自适应**：不需要手动指定衰减时机
2. **与验证集挂钩**：基于验证准确率调整，符合"选最佳模型"的目标
3. **结合早停**：patience 参数同时起到了防止过拟合的作用

## 九、实践建议

### 如何设置学习率

| 场景 | 建议初始 lr |
|------|-------------|
| Adam 优化器 | 1e-3 ~ 1e-4（常用 0.001） |
| SGD 优化器 | 0.1 ~ 0.01 |
| 小模型 / 浅层网络 | 可适当降低（如 5e-4） |
| 大模型 / 深层网络 | 可从 1e-4 开始 |

### 如何设置衰减参数

| 参数 | 建议值 | 说明 |
|------|--------|------|
| factor | 0.5 或 0.1 | 每次减半或减到 1/10 |
| patience | 5 ~ 10 | 波动大则设大（如 10） |
| min_lr | 1e-6 | 最低下限 |

### 本实验参数总结

```
初始 lr    = 0.001
factor     = 0.5    (每次减半)
patience   = 5      (5个epoch不上升则衰减)
min_lr     = 1e-6
最终 lr    = 0.00025
衰减次数   = 2次
```

## 十、总结

| 要点 | 说明 |
|------|------|
| 学习率是"步长" | 太大震荡，太小收敛慢 |
| 衰减指标 | 验证集损失 val_loss（mode='min'），不是准确率 |
| 衰减时机 | ReduceLROnPlateau 自动判断（patience 控制） |
| 衰减效果 | Epoch 28: val_loss停滞触发，Epoch 46: 再次停滞触发 |
| 最终结果 | 测试准确率 74.61%，与验证集最佳 74.92% 非常接近 |
| 核心思想 | 先大步探索，再小步精细调整 |

**一句话理解**：学习率衰减就是"先快跑接近目标，再小步走到精确位置"——当验证集损失连续多个 epoch 不再下降时，自动缩小步长，让模型精细微调。
