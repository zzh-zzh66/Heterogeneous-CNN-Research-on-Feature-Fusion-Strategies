## 一、卷积层

**卷积层 = 一组过滤器在图像上滑动，做特征提取。**

用项目代码中的一层举例（[branch1.py](file:///E:\人工智能论文\src\models\branch1.py)）：

```python
nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3, padding=1)
```

```
输入: 3通道图像 (RGB)          32个过滤器
   ┌──────────────┐              ┌────────┐
   │   32×32×3    │   ───────►   │ 每个是 │
   │              │              │ 3×3×3  │
   └──────────────┘              └────────┘ × 32 个
                                         ↓
                                   输出: 32×32×32
                                   (32个特征图)
```

### 每一步在做什么？

```
输入图像的一个 3×3 局部区域：

   R  G  B          一个过滤器（以检测"垂直边缘"为例）：
 ┌──────┐          ┌────────────┐
 │ 1  2 3 │          │ -1  0  +1 │   ← 权重在训练中学习得到
 │10 11 9 │   ×     │ -2  0  +2 │   ← 这个过滤器对"垂直边缘"响应最强
 └──────┘          └────────────┘
    ↓
卷积计算：(1×-1 + 2×0 + 3×1) + (10×-2 + 11×0 + 9×2) = 某数值
```

过滤器在图像上逐格滑动，每一步输出一个数值，所有输出拼成一张**特征图**。

### 为什么需要多层卷积？

```
原图 → Conv1(3→32) → Conv2(32→64) → Conv3(64→64)
 ↓
32种基础纹理         组合成更复杂的模式         提炼出猫的纹理模式
(点、短线)              (边缘、角)                 (具体部位)
```

**每一层卷积**，都是在**前一层的输出上继续抽象**：

| 层 | 输入 | 输出 | 提取什么 |
|----|------|------|---------|
| Conv1 | 像素 | 32种基础纹理 | 边缘方向 |
| Conv2 | 32种纹理 | 64种更复杂特征 | 纹理组合 |
| Conv3 | 64种特征 | 64种局部部件 | 局部形状 |

### 为什么叫"卷积"？

这是一种数学运算的名字——**"卷积"**。但可以简单理解为"在图像上用一个滑动窗口做加权求和"：

```
滑动窗口 3×3          每滑动一步，输出一个值
    ┌───┬───┬───┐
    │ w1│ w2│ w3│      output = Σ (window[i] × kernel[i])
    ├───┼───┼───┤
    │ w4│ w5│ w6│
    ├───┼───┼───┤
    │ w7│ w8│ w9│
    └───┴───┴───┘
      ↓
   过滤器权重（在训练中自动学习得到）
```

**一句话总结**：卷积层 = 一组可学习的过滤器，在图像上滑动提取特征。多层叠加 = 从"像素"逐渐抽象到"语义概念"。

## 二、卷积核 vs 过滤器

**卷积核和过滤器是同一个东西的两种叫法。**

| 叫法 | 来源 | 强调 |
|------|------|------|
| **卷积核** (kernel) | 数学中的卷积运算 | 它是一个"核"矩阵，在图像上滑动做卷积 |
| **过滤器** (filter) | 信号处理 | 它"过滤"出特定模式 |

一个 3×3 卷积核（过滤器）在输入图像上滑动：

```
输入图像 5×5          卷积核 3×3          输出 3×3
┌───────┐            ┌───┐               ┌───┐
│1 2 3 4│            │1 0│               │·  │
│5 6 7 8│    *       │0 1│    =           │·  │
│9 0 1 2│            └───┘               │·  │
│3 4 5 6│                                └───┘
└───────┘
   ↑                                            ↑
 滑动窗口每步计算：1×1+2×0+5×0+6×1 = 7  →  填入输出
```

在代码中，`nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3)` 就是创建了 32 个 3×3×3 的卷积核（过滤器）。

## 三、通道数的含义与变化

### 通道数 = 过滤器数量

以项目中的 Branch1 为例（[branch1.py](file:///E:\人工智能论文\src\models\branch1.py)）：

```
输入图像: [32×32×3]    3 通道 = R, G, B
              ↓
  Conv1: 3→32 通道      32 个过滤器（每个 3×3×3）
              ↓
  特征图: [32×32×32]   32 通道 = 32 种"纹理检测结果"
              ↓
  Conv2: 32→64 通道     64 个过滤器（每个 3×3×32）
              ↓
  特征图: [32×32×64]   64 通道 = 64 种更复杂的特征
              ↓
  Conv3: 64→64 通道     64 个过滤器（每个 3×3×64）
              ↓
  特征图: [16×16×64]   池化后尺寸减半，通道不变
```

### 每个过滤器在做什么？

一个过滤器就是一个**特征检测器**。以 Conv1 的 32 个过滤器为例：

| 过滤器 # | 可能检测 | 输出一个通道 |
|----------|---------|:----------:|
| 过滤器 1 | 水平边缘 `—` | 通道 1 |
| 过滤器 2 | 垂直边缘 `\|` | 通道 2 |
| 过滤器 3 | 45° 斜边 `/` | 通道 3 |
| ... | ... | ... |
| 过滤器 32 | 某种颜色斑点 | 通道 32 |

每个过滤器是一个 `3×3×3` 的小矩阵，在图像上滑动做卷积。输出是一个二维的"激活图"——亮点表示"这里检测到了该特征"。

### 为什么通道数会改变？

```
输入 3 通道 → 输出 32 通道
```

**3 → 32 的本质**：原来只有 3 种信息（红绿蓝），现在用 32 个过滤器，把图像分解成了 32 种"基础纹理模式"。

**32 → 64 的本质**：用 64 个过滤器组合前面的 32 种基础纹理，检测更复杂的模式（比如"水平边缘+斑点"组合）。

## 四、不同大小卷积核的区别（3×3 vs 5×5 vs 7×7）

结合项目中三个分支的设计：

| 卷积核 | 参数 | 每个神经元"看"的区域 | 类比 |
|:---:|:---:|:---|:---|
| 3×3 | 9 个权重 | 3×3 = 9 像素 | 用放大镜看细节 |
| 5×5 | 25 个权重 | 5×5 = 25 像素 | 正常阅读距离 |
| 7×7 | 49 个权重 | 7×7 = 49 像素 | 站远一点看整体 |

```
同一张 32×32 图像，不同卷积核的"视野"：

3×3 卷积核                        7×7 卷积核
┌──────────────┐                 ┌──────────────┐
│ · · · · · ·  │                 │ ┌────────┐ · │
│ · ┌───┐ · ·  │                 │ │ 看到整  │ · │
│ · │看这│ · ·  │                 │ │ 个猫脸  │ · │
│ · └───┘ · ·  │                 │ └────────┘ · │
│ · · · · · ·  │                 │ · · · · · ·  │
└──────────────┘                 └──────────────┘
 只看到猫耳朵的一小撮毛              看到整张猫脸 + 部分背景
```

**综合起来看**：小卷积核 → 精细局部纹理（毛发的方向），大卷积核 → 整体轮廓（猫脸的形状）。项目的异构融合模型正是把三种视野组合起来，既有细节又有全局。

## 五、浅层、中层、深层如何区分？

**不是通过通道数，也不仅仅靠卷积核大小，而是通过"感受野的大小"来区分的。** 感受野由两个因素共同决定：

```
感受野 = 卷积核大小 + 网络层数（卷积层数越多，感受野累计越大）
```

项目中三个分支的对比：

| 分支 | 卷积核 | 层数 | 最终感受野 | 关注层级 |
|------|:---:|:---:|:---:|:---:|
| [Branch1](file:///E:\人工智能论文\src\models\branch1.py) | 3×3 | 3 层 | **7×7** | 浅层纹理 |
| [Branch2](file:///E:\人工智能论文\src\models\branch2.py) | 5×5 | 4 层 | **17×17** | 中层形状 |
| [Branch3](file:///E:\人工智能论文\src\models\branch3.py) | 7×7 | 5 层 | **26×26 → 32×32** | 深层语义 |

```
感受野从小到大：

Branch1 (7×7)          Branch2 (17×17)        Branch3 (26×26→全局)
    ┌─┐               ┌─────┐               ┌──────────┐
    │毛│               │ 耳朵 │               │  整只猫   │
    └─┘               └─────┘               └──────────┘
   纹理细节              部件形状                整体语义
```

**一句话**：大卷积核 + 多卷积层 = 大感受野 = 深层语义。通道数（32→64→128）只决定"能装多少种特征"，和浅/中/深没有直接关系。

## 六、Grad-CAM 热力图解读

### 热力图中的亮点是什么？

**不是"不同过滤器检测到相同特征"，而是"所有过滤器对最终决策的综合贡献"。**

项目中的 Grad-CAM 实现（[visualization.py#L347-L420](file:///E:\人工智能论文\src\utils\visualization.py#L347-L420)）工作原理：

```
步骤 1: 取最后一层卷积的所有通道（64个特征图）
         ↓
步骤 2: 对每个通道求梯度 → 得到该通道对"预测结果"的重要性权重
         ↓
步骤 3: 加权求和：64个通道 × 各自权重 → 合并成 1 张热力图
         ↓
步骤 4: 热力图叠加到原图 → 红色/黄色亮点 = 模型做决策时最关注的区域
```

### 热力图解读

```
┌──────────────────┐
│  🟡🟡            │  黄色 = 高关注 → 模型主要看这里
│ 🟡🟡🟡           │
│  🟡🟡    🔵🔵🔵  │  蓝色 = 低关注 → 模型不太在乎
│       🔵🔵🔵      │
└──────────────────┘
```

**热力图说明的是**：模型最终判断"这是猫"时，主要依赖图像的**哪个空间位置**，而不是哪个过滤器。它把所有通道的重要性加权求和，最终告诉你模型在"看图片的哪里"来做分类决策。

## 七、项目中三个分支的设计总结

| 分支 | 感受野 | 关注层级 | 适合检测 |
|------|:------:|:--------:|---------|
| Branch1 (3×3, 3层) | 7×7 | 浅层 | 纹理细节、边缘方向 |
| Branch2 (5×5, 4层) | 17×17 | 中层 | 部件形状、局部结构 |
| Branch3 (7×7, 5层) | 全局 | 深层 | 整体语义、类别判别性信息 |

异构融合模型（HeteroFusion）的核心思想就是：将这三个不同感受野分支的输出拼接（cat），让模型同时拥有局部细节和全局语义的感知能力。
